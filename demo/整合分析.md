引擎里所有涉及「角色」的数据源，现在还分散在三个地方：

## 数据源现状

| 位置 | 存储什么 | 管理方式 |
|------|---------|---------|
| **① JSON 角色卡**（`voice_cards/default/*.json`） | 声纹 + 性格 + 做事风格 | ✅ 已被 character 模块管理 |
| **② DB characters 表**（`novel.db`） | 身份 + 关系 + 弧线 + 别名 + 标签 | ✅ 已被 character 模块管理 |
| **③ Story Contract 内存文件**（`.story/memory/characters.json`） | 故事合同中的角色描述 | ❌ 未被 character 模块管理 |
| **④ 章节计划表**（DB `chapter_plans.character_focus`） | 本章聚焦哪几个角色 | ❌ 未被 character 模块管理 |
| **⑤ 角色声纹导出/导入** | 跨项目搬角色卡 | ❌ 未被 character 模块管理 |

## 可以往 character 模块加的功能

### 1. character story-init → 角色进入故事合同
当前 `story init` 只创建空的 `characters.json`。需要一个命令把已配置好的角色卡数据写入故事合同系统，让合同构建时自动引用角色卡的身份/关系/弧线。

```
python novel.py character sync-story [角色名]
```

### 2. character focus → 标记章节聚焦角色
在 `character show` 中增加"最近出场章节"追踪，或者支持设置某角色当前处于"故事线活跃/暂离/已退场"状态。

### 3. character import/export → 跨项目搬卡
```
python novel.py character export 韩烈  → 韩烈.json
python novel.py character import 韩烈.json
```

### 4. character relationship graph → 关系图谱展示
现有的 `relationship` 字段只是文本字符串。可以加一个 `relate` 子命令专门管理关系：

```
python novel.py character relate 韩烈 苏晚晴 知己
python novel.py character relate 韩烈 张天师 对立
python novel.py character relation-graph
```

### 5. character arc check → 弧线进度检查
对比当前写作进度和角色的 `arc` / `motivation`，检查角色是否在应有的弧线阶段。

### 6. 旧 Voice Profile 系统（遗留）
`scripts/voice_profile_loader.py` 和 `scripts/export/import_voice_profiles.py` 是老版本留下的声纹配置系统，角色卡的 JSON 文件已经取代了它们。可以考虑废弃或迁移。
